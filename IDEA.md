# IDEA.md

Usa este archivo para capturar una idea vaga de MVP. Puedes dejar campos incompletos. El agente debe inferir lo razonable cuando falte informacion y solo preguntar cuando algo sea realmente bloqueante.

## Nombre provisional

Ejemplo: "Tutor de histogramas", "Generador de guiones cortos", "Priorizador de leads".

## Idea en una frase

Describe el MVP en una sola frase.

Ejemplo: "Una herramienta que ayuda a equipos comerciales a priorizar leads usando datos simples y explicaciones accionables".

## Problema que resuelve

Que dolor concreto tiene el usuario hoy.

Ejemplo: "El equipo revisa muchos leads manualmente y no sabe cuales atender primero".

## Usuario objetivo

Quien usaria el MVP primero.

Ejemplo: "Coordinadores comerciales de equipos B2B pequenos".

## Resultado esperado

Que cambio observable debe lograr el usuario despues de usar el MVP.

Ejemplo: "Tener una lista priorizada de leads con razones claras en menos de 10 minutos".

## Señal de exito inicial

Como sabremos que el MVP valio la pena en la primera prueba.

Ejemplo: "3 de 5 usuarios pueden completar el flujo principal sin ayuda y consideran util el resultado".

## Contexto

Incluye datos, proceso actual, herramientas existentes, dominio, restricciones de negocio o historia de la idea.

## Plataforma deseada

Ejemplos:

- Web app simple.
- CLI interna.
- Documento automatizado.
- Dashboard local.
- Extension futura.

## Entrada y salida esperadas

Describe que informacion entra al MVP y que resultado debe salir.

Ejemplo:

- Entrada: CSV con leads.
- Salida: top 5 leads con razones de prioridad.

## Restricciones

Ejemplos:

- Sin login en el MVP.
- Sin pagos.
- Sin integraciones externas al inicio.
- Debe funcionar con CSV.
- Debe poder revisarse por una persona antes de tomar decisiones.

## Inspiraciones

Productos, flujos o experiencias similares.

Ejemplo: "Google Sheets por simplicidad, Linear por claridad visual, ChatGPT por asistencia".

## Lo que NO debe incluir el MVP

Define limites explicitos.

Ejemplo:

- No app movil.
- No marketplace.
- No modelo predictivo entrenado desde cero.
- No automatizacion irreversible.

## Preguntas abiertas

Lista dudas que todavia existen.

Ejemplo:

- Que datos reales estaran disponibles?
- Quien aprueba el resultado final?
- Que metrica define exito en la primera semana?

## Decisiones bloqueantes conocidas

Usa esta seccion solo para decisiones que impiden avanzar. Si no hay ninguna, escribe "Ninguna por ahora".

Ejemplo: "No sabemos si se permite usar datos reales de clientes".

## Supuestos permitidos

Autoriza inferencias razonables.

Ejemplo:

- Si no hay usuario definido, asumir el usuario mas cercano al problema.
- Si no hay plataforma definida, asumir web app o documento accionable.
- Si falta metrica, proponer una metrica de adopcion y una de resultado.
- Si hay mucho alcance, reducir a una vertical slice construible en una semana.

## Nivel de incertidumbre aceptado

Marca una opcion:

- Bajo: el agente debe apegarse mucho a lo escrito.
- Medio: el agente puede inferir alcance y metricas no criticas.
- Alto: el agente debe proponer una version razonable aunque falten datos.

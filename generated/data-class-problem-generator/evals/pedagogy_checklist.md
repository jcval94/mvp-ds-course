# Pedagogy Checklist

## Experiencia completa

- [ ] El concepto tiene definición breve.
- [ ] El concepto tiene intuición visual.
- [ ] El concepto tiene error común documentado.
- [ ] El concepto tiene visualización obligatoria.
- [ ] Los tres modos usan el mismo `visualSpec`.

## Aprender

- [ ] El módulo empieza con intuición.
- [ ] Las secciones son breves.
- [ ] Hay mini-checkpoint.
- [ ] Hay transición a práctica.

## Ejercitar

- [ ] El ejercicio tiene rol y contexto.
- [ ] La pregunta evalúa el concepto.
- [ ] Los distractores son plausibles.
- [ ] El feedback enseña.

## Enseñar en vivo

- [ ] El profesor recibe guion minuto a minuto.
- [ ] El prompt para Codex es ejecutable.
- [ ] El prompt para Gemini anticipa errores.
- [ ] El notebook blueprint tiene estructura útil.
- [ ] Hay plan offline.

## Fallar si

- La visualización es decorativa.
- El alumno puede responder sin mirar el visual.
- La clase en vivo depende de ejecutar IA dentro del navegador.

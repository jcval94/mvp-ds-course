# Practice Case Quality Checklist

- [ ] El caso tiene historia y rol.
- [ ] La pregunta evalúa el concepto seleccionado.
- [ ] La visualización es adecuada.
- [ ] Hay badge de concepto evaluado.
- [ ] Hay badge de visualización obligatoria.
- [ ] Los distractores son plausibles.
- [ ] Cada opción tiene feedback útil.
- [ ] Hay pistas o evidencia visual.
- [ ] Se puede copiar JSON.
- [ ] El nivel de dificultad es coherente.

## Fallar si

- La pregunta no depende del visual.
- Hay más de una respuesta correcta.
- El feedback solo dice correcto/incorrecto.

# Project Structure

```text
generated/data-class-problem-generator/
├── README.md
├── IDEA.md
├── docs/
├── app/
│   └── index.html
├── examples/
└── evals/
```

## Propósito

- `README.md`: explicación operativa del MVP.
- `IDEA.md`: cambio conceptual y restricciones.
- `docs/`: arquitectura documental del producto.
- `app/index.html`: app autocontenida con los tres modos.
- `examples/`: ejemplos JSON exportables.
- `evals/`: rúbricas para módulo, práctica, clase en vivo y visuales.

## Nota histórica

El proyecto ya no depende de `reference/demos`. Si aparecen archivos históricos en esa ruta, pueden usarse como inspiración visual, pero no bloquean desarrollo ni QA.

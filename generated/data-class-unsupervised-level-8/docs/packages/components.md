# Paquete: Componentes

## Supuestos

- Audiencia adulta principiante; español mexicano cálido.
- Historia aprobada: `docs/stories/LEVEL_8.md`; escena `L8-S06`.
- Aprender y Ejercitar usan datos sintéticos narrativos; En vivo usa `Palmer Penguins`.
- El narrador se renderiza como subtítulo y Don Juan conserva la decisión del negocio.

## ConceptSpec

- **Nivel/bloque:** 8, Reducción dimensional.
- **Objetivo:** Leer componentes mediante sus cargas.
- **Definición:** Una componente es una combinación lineal de variables originales.
- **Intuición:** El visual hace visible vectores de carga sobre ejes nuevos antes de resumirlo.
- **Error plausible:** Convertir una salida exploratoria o una métrica aislada en una decisión automática.
- **Mecanismo visual:** vectores de carga sobre ejes nuevos.
- **Estados:** Antes de decidir → Evidencia revisada.
- **Unidad:** una observación es una noche operativa, nunca una persona.
- **Variables:** cargas_pc1, cargas_pc2.
- **Límite:** Clusters y anomalías son hipótesis para revisión humana; no prueban tipos naturales, fraude ni causalidad.

## LearningModule

1. El primer eje mezcla volumen, espera y merma.
2. Ejecutar **Comparar estados de componentes**.
3. Cita una marca visible, responde la pregunta de Don Juan y nombra un límite.

## PracticeExercise

- **Guiado:** Observa cargas de variables y cita una marca visible. ¿Qué conclusión guiada sobre componentes conserva el alcance?
- **Transferencia:** Observa cargas de variables y cita una marca visible. ¿Qué exige transferir componentes a otra decisión?
- **Bloqueo:** 1 cambios y todas las marcas requeridas.
- **Separación:** Aprender revela el mecanismo; los dos ejercicios usan incidentes y evidencia nuevos.

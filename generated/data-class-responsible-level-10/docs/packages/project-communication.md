# Paquete: Comunicación del proyecto

## Supuestos

- Audiencia adulta principiante; español mexicano cálido.
- Historia aprobada: `docs/stories/LEVEL_10.md`; escena `L10-S17`.
- Aprender y Ejercitar usan datos sintéticos narrativos; En vivo usa `Wine Quality · UCI`.
- El narrador se renderiza como subtítulo y Don Juan conserva la decisión del negocio.

## ConceptSpec

- **Nivel/bloque:** 10, Mini-proyecto.
- **Objetivo:** Entregar acción, evidencia, incertidumbre y siguiente revisión.
- **Definición:** Comunicar cierra con acción, incertidumbre y siguiente revisión.
- **Intuición:** El visual hace visible brief de decisión responsable antes de resumirlo.
- **Error plausible:** Convertir una salida exploratoria o una métrica aislada en una decisión automática.
- **Mecanismo visual:** brief de decisión responsable.
- **Estados:** Antes de decidir → Evidencia revisada.
- **Unidad:** una observación es una celda semanal agregada con al menos 25 elegibles; no hay registros individuales.
- **Variables:** periodo_semana, ofrecidos, completados.
- **Límite:** Los grupos son categorías de auditoría consentidas y agregadas; ninguna tabla infiere identidad, intención ni rasgos personales.

## LearningModule

1. Paco entrega una apertura limitada de un solo local.
2. Ejecutar **Comparar estados de comunicación del proyecto**.
3. Cita una marca visible, responde la pregunta de Don Juan y nombra un límite.

## PracticeExercise

- **Guiado:** Observa acción, evidencia, incertidumbre y revisión y cita una marca visible. ¿Qué conclusión guiada sobre comunicación del proyecto conserva el alcance?
- **Transferencia:** Observa acción, evidencia, incertidumbre y revisión y cita una marca visible. ¿Qué exige transferir comunicación del proyecto a otra decisión?
- **Bloqueo:** 1 cambios y todas las marcas requeridas.
- **Separación:** Aprender revela el mecanismo; los dos ejercicios usan incidentes y evidencia nuevos.

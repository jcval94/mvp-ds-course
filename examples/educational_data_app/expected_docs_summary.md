# Expected Docs Summary

## Product Brief esperado

El MVP debe enfocarse en una experiencia guiada para entender histogramas. El usuario inicial puede ser un docente que lo usa en clase o un estudiante que practica solo.

## PRD esperado

Funcionalidades MVP:

- Mostrar dataset incluido.
- Renderizar histograma.
- Permitir cambiar cantidad de bins.
- Mostrar narrativa breve sobre forma, centro y dispersion.
- Incluir una pregunta de reflexion.

Post-MVP:

- Guardar progreso.
- Crear datasets propios.
- Integracion LMS.

## Vertical slice esperada

Abrir una pagina, ver un histograma de edades o calificaciones, cambiar bins y recibir una explicacion textual.

## Skills esperadas

- `mvp-product-strategist`
- `prd-writer`
- `eval-designer`
- `qa-reviewer`

## Evals esperados

Debe aprobar si mantiene una sola leccion interactiva. Debe fallar si propone LMS, cuentas de estudiantes o curso completo.

## Riesgos esperados

- Explicaciones demasiado largas.
- Confusion entre histograma y barras.
- Alcance educativo demasiado amplio.

# Expected Docs Summary

## Product Brief esperado

El MVP prioriza leads con una puntuacion explicable basada en campos simples como industria, tamaño, ultima actividad y valor potencial.

## PRD esperado

Funcionalidades MVP:

- Subir o usar CSV de ejemplo.
- Validar columnas minimas.
- Calcular score por reglas.
- Mostrar top leads con razones.
- Exportar resultado.

Post-MVP:

- Integracion CRM.
- Roles y permisos.
- Modelo predictivo.
- Historial de cambios.

## Vertical slice esperada

Cargar CSV de 20 leads, calcular score, ver top 5 y exportar recomendaciones.

## Skills esperadas

- `mvp-product-strategist`
- `prd-writer`
- `harness-designer`
- `qa-reviewer`

## Evals esperados

Debe aprobar si usa reglas explicables y CSV. Debe fallar si requiere CRM, modelo predictivo entrenado o permisos complejos.

## Riesgos esperados

- Datos incompletos.
- Scoring percibido como arbitrario.
- Alcance de dashboard demasiado amplio.

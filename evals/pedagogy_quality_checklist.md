# Pedagogy Quality Checklist

## Aprender

- [ ] Activa conocimiento previo.
- [ ] Presenta intuición antes del formalismo.
- [ ] El visual representa el mecanismo.
- [ ] La interacción modifica una variable significativa.
- [ ] Incluye comparación o predicción.
- [ ] Corrige un error común.
- [ ] Tiene checkpoint con explicación.

## Ejercitar

- [ ] Desde Nivel 2 hay ejercicio guiado y ejercicio de transferencia.
- [ ] El rol, protagonista, contexto y presión realista sirven al objetivo.
- [ ] La historia es distinta de Aprender y no revela la respuesta por sí sola.
- [ ] La animación revela evidencia antes de habilitar la respuesta.
- [ ] El `evidenceContract` declara pasos, `evidenceIds` y desbloqueo.
- [ ] La respuesta se habilita solo al completar el contrato.
- [ ] La evidencia es necesaria.
- [ ] La respuesta correcta es defendible.
- [ ] Los distractores son plausibles.
- [ ] Cada opción tiene feedback específico.
- [ ] Las pistas son graduadas.
- [ ] La conclusión transfiere el aprendizaje.

## Enseñar en vivo

- [ ] El guion tiene tiempos.
- [ ] El dataset es snapshot público real reproducible con fuente, licencia, fecha y SHA-256.
- [ ] El modo docente está oculto por defecto y aclara que no es autenticación real.
- [ ] La demo puede ejecutarse o simularse.
- [ ] Hay preguntas socráticas.
- [ ] Hay evaluación rápida.
- [ ] Existe plan offline.
- [ ] Codex tiene un rol técnico y Gemini/ChatGPT un rol de facilitación o revisión.
- [ ] Hay verificación humana y regla de privacidad.
- [ ] Otro docente puede usar el paquete con ajustes menores.

## Fallar si

- La respuesta se obtiene sin mirar el visual.
- La interacción solo cambia estilo.
- El QA solo comprueba una diferencia de HTML.
- El visual cita marcas que no existen en pantalla.
- El feedback se limita a correcto o incorrecto.
- La narrativa afirma más de lo que muestran los datos.
- En vivo usa dataset sintético como fuente principal.
